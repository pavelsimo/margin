Figure 1 — R-SWA at a glance: each new token sees every reference (visual) token plus only the last n = 128 output tokens, keeping the KV cache flat for the whole decode. Reference tokens never enter state transitions, so image features stay crisp — the failure mode of vanilla SWA.

1 Introduction

Humans handle long-horizon parsing with ease — copying out a book across hundreds of pages, or transcribing hours of audio — yet these are exactly the tasks where models break down. No current end-to-end OCR model parses even ten pages in one forward pass; documents are fed page-by-page in a loop, with memory wiped at every step. That loop is an external scheduler stitching together fragments of what should be one continuous process — a workaround, not intelligence.

The way people actually transcribe suggests a different attention pattern. A copyist never re-reads everything already written; they watch the source and glance at the last few characters to stay oriented. This is not full attention — the history is never fully consulted — and not linear attention either, since the source must not be blurred by recurrent state updates. R-SWA is built to match this natural flow. The contributions:

  • R-SWA — each token attends to all reference tokens (visual tokens and the prompt) plus a causal window over the preceding 128 output tokens, keeping the KV cache constant during inference.
  • Unlimited OCR — DeepSeek OCR with every decoder attention layer replaced by R-SWA: dozens of pages parsed in one pass, and 93% on OmniDocBench v1.5, 6 points above the baseline.
  • A first validation that reference-based, linear-cost attention is lossless for parsing — with ASR and translation as natural next applications.

2

2 Related Works

Pipeline systems chain a layout detector with block-level recognizers, bridged by cropping and rectification heuristics; LLM decoders have modernized rather than replaced this recipe. End-to-end VLMs instead parse a whole page in a single forward pass, which raises the bar for the decoder but makes architectural progress directly transferable to general-purpose models.

Two components bound an end-to-end model: the encoder’s compression ratio, which sets prefill length, and the decoder’s KV cache — which in every current model grows with each output token. That growth is the bottleneck this work removes.

3 Methodology

3.1 Long-horizon Parsing

When copying a book by hand, attention rests on three things: the source, the last few written characters, and the next one to write. Nothing like a full transcript is held in mind — distant output fades away. This soft forgetting appears to be what keeps cognitive load flat over hours of transcription, and it is the behavior Unlimited OCR reproduces.

figure 2 — framework: encoder, MoE decoder, queued KV cache

Figure 2 — The framework: DeepEncoder compresses input pages 16× into visual tokens; a 3B-MoE decoder (≈570M active) runs R-SWA in every layer. The KV cache is a queue of capacity m + n — each new token evicts the oldest output entry, so memory never grows during generation.

3

3.2 Architecture

Unlimited OCR keeps DeepSeek OCR’s encoder–decoder layout: DeepEncoder feeding a 3B-parameter MoE LLM with roughly 500M activated parameters. The single change is decoder-side — vanilla multi-head attention is swapped for R-SWA in all layers, adding a fixed-width output KV buffer of width n to the reference cache of size m.

3.3 DeepEncoder

DeepEncoder cascades SAM-ViT (window attention) into CLIP-ViT (global attention) with 16× token compression at the bridge, turning a 1024×1024 page into 256 visual tokens. Two resolution modes are kept: “Base” for multi-page inputs and “Gundam” for single pages. Visual tokens are encoded once and stay static for the whole parse — a property R-SWA depends on.

3.4 Reference Sliding Window Attention

Assume one visual token decodes to roughly ten text tokens: 10K visual tokens — 20–30 pages — demand an output beyond 100K tokens, untenable for vanilla attention. R-SWA constrains each token’s view to a two-segment window:

N(t) = P ∪ Dₙ(t), P = {1, …, Lₘ}
C_MHA(T) = Lₘ + T  vs.  C_R-SWA(T) = Lₘ + min(n, T) ≤ Lₘ + n

The prefix window m is fixed by page count; the decode window n = 128 slides causally. The cache ratio (Lₘ + n) / (Lₘ + T) shrinks toward zero as output grows — the longer the parse, the larger the saving.

4 Experimental Settings

Training uses ~2M document OCR samples at a 9:1 single- to multi-page ratio; multi-page examples are synthesized by concatenating single pages (2–50, separated by an <page> tag), everything packed to 32K tokens. Starting from the DeepSeek OCR checkpoint, the LLM is continue-trained for 4,000 steps (batch 256, AdamW, cosine schedule, lr 1e-4) with DeepEncoder frozen. R-SWA cache management is implemented in both Transformers and SGLang.

5 Evaluation

OmniDocBench is the main benchmark — v1.5 for comparison with classic models including the baseline, v1.6 against current SOTA. Metrics span text edit distance, formula CDM, table TEDS, and reading-order edit distance.

5.4 Long-horizon Parsing

With a fixed KV cache, Unlimited OCR prefills tens of pages and parses from first to last in one pass at constant latency. Remaining errors trace mostly to small text at the 1024×1024 “Base” resolution, not to R-SWA losing its place.

6 Efficiency Analysis

Under ideal concurrency the two models start equal at 256 output tokens, but the baseline’s TPS decays with length while Unlimited OCR holds steady — a 35% advantage by 6K tokens, and 12.7% end-to-end on OmniDocBench (5,580 vs 4,951 TPS).

7 Limitation and Future Work

Parsing is not yet truly unlimited: prefill still grows with page count under the 32K context. Short term, longer-context training (128K); long term, a prefill pool the model fetches KV chunks from — the machine version of flipping pages. R-SWA is also slated for ASR and translation.

8 Conclusion

Replacing every decoder attention with reference-based causal SWA proves lossless for parsing: useful history flows forward through the window, and soft forgetting of one’s own output — the human strategy — removes attention computation and memory as the bottleneck for long-horizon work.
